## 严重问题 (Critical)

### 1. WebSocket 运行事件在部署环境完全失效

- **文件:** `frontend/lib/api.ts:572-584`, `frontend/next.config.ts:7-19`
- **现象:** `getRunEventsWsUrl()` 在 `NEXT_PUBLIC_API_BASE_URL=/api`（部署默认值）时返回空字符串。Next.js standalone 的 rewrite 不支持 WebSocket upgrade。
- **影响:** 部署后实时运行事件流（run events streaming）功能完全不可用。

### 2. 前端 systemd 服务 WorkingDirectory 错误

- **文件:** `deploy/systemd/blueprint-re-frontend.service:8-9`
- **现象:** `WorkingDirectory` 指向 `__FRONTEND_RELEASE_DIR__/frontend`，但 Next.js standalone 输出的 `server.js` 在上级目录。
- **影响:** `node server.js` 启动时找不到模块，前端服务无法启动。

### 3. Manager Agent 在 API Key 未配置时静默失败

- **文件:** `manager-agent/src/server.js:17,2181`; `scripts/deploy_user_systemd.sh:275`
- **现象:** 健康检查 `/healthz` 返回 200 OK，但所有 `/chat` 和 `/chat-stream` 请求抛出 "API key not configured" 错误。部署脚本默认写入空值。
- **影响:** Manager Agent 进程正常运行但无法处理任何请求，问题难以排查。

### 4. Run 状态在子进程创建前写入（竞态窗口）

- **文件:** `backend/app/services/worker_service.py:714-731`
- **现象:** run 状态在第 716-717 行设为 "running" 并发出 `run_started` 事件，而 `subprocess.Popen` 直到第 723-731 行才执行。若进程创建失败，异常处理（第 983 行）会修正状态为 "failed"，但之间存在不一致窗口。`cancel_run`（第 452 行）在此期间调用会因 `self._processes` 尚未设置而静默成功。
- **影响:** 进程可能脱离监控继续运行，状态记录不准确。

### 5. 输出文件缺失被静默忽略（验证漏洞）

- **文件:** `backend/app/services/executor_validation_service.py:158-164`
- **现象:** manifest 声明的 `created_assets` 在磁盘上不存在时，代码执行 `continue` 跳过，不产生 `ValidationIssue`。AI reviewer 可能捕获此问题，但确定性验证关会误报通过。
- **影响:** 执行器声称产出了文件但实际未写入磁盘的情况不会被检测到。

### 6. 资产与输出合约按位置 zip 配对错误

- **文件:** `backend/app/services/worker_service.py:1760-1762`
- **现象:** `zip(assets, manifest_created_assets, strict=False)` 按位置而非按 key 配对。`strict=False` 在列表长度不一致时静默截断。若两个列表顺序不同，`real_by_planned` 映射会把错误的资产 ID 绑定到输出合约上。
- **影响:** 静默数据错配，输出合约的 `asset_id` 和 `status` 回写错误。

### 7. delete_project 与后台执行线程竞态

- **文件:** `backend/app/services/project_service.py:202-209`
- **现象:** `shutil.rmtree(root)` 删除项目目录时，后台线程可能在 `_execute_run` 中正在读写同一目录下的文件。
- **影响:** 后台线程遭遇 `FileNotFoundError` / `OSError`，可能导致部分状态丢失或损坏。

### 8. URL 解析可导致 Server 进程崩溃

- **文件:** `manager-agent/src/server.js:2522`
- **现象:** `new URL(req.url, ...)` 在 `req.headers.host` 包含非法字符时抛出 `TypeError`，发生在 `handle()` 函数顶层 try-catch 之外。`createServer` 回调使用 `void handle(req, res)` 丢弃了 promise rejection。
- **影响:** 单个恶意请求可导致整个 Manager Agent 进程退出。

## 高危问题 (High)

### 9. get_run 对不存在的 run_id 返回 500 而非 404

- **文件:** `backend/app/api/runs.py:90-93`
- **现象:** `next(item for item in graph.runs if item.run_id == run_id)` 无默认值，`StopIteration` 被 FastAPI 转为 500。
- **修复:** 改为 `next((...), None)` 并检查 None 后返回 404。

### 10. WebSocket 在 run 完成后永不主动关闭

- **文件:** `backend/app/api/runs.py:166-184`
- **现象:** run 到达终态且所有事件已发送后，循环继续 sleep 0.25s 并无限轮询，无 `break`。直到客户端断开才释放。
- **影响:** 每个 WebSocket 连接在 run 结束后仍占用一个协程，持续消耗资源。

### 11. 事件来源前缀匹配导致归属错误

- **文件:** `backend/app/services/worker_service.py:1169-1171`
- **现象:** `"run_started"` 和 `"run_cancelled"` 等 manager 发出的事件因以 `"run_"` 开头被误标为 `source="executor"`。
- **影响:** 事件来源追踪不准确，影响调试和审计。

### 12. _run_status 未加锁读取 graph 状态

- **文件:** `backend/app/services/worker_service.py:1660-1662`
- **现象:** 后台线程调用 `_run_status` 时不持有项目锁。返回的状态可能来自部分更新的 graph。
- **影响:** `_execute_run` 中基于此状态的控制流决策（取消检查等）可能做出错误判断。

### 13. 脆弱的 HTTPResponse 私有属性访问

- **文件:** `backend/app/services/manager_service.py:350-357`
- **现象:** 访问 `upstream.fp.raw._sock` 等私有属性来设置 socket 超时。内部结构在不同 Python 版本和 SSL/非 SSL 连接下不同。
- **影响:** 静默失败时上游读操作可能无限挂起，导致流式聊天响应卡死。

### 14. compact() 参数顺序疑似错误

- **文件:** `manager-agent/src/server.js:2044-2051`
- **现象:** 两个显式 `undefined` 占位符将 `signal` 和 `thinkingLevel` 推到了靠后的参数位置。若函数签名为 `(preparation, model, apiKey, signal, thinkingLevel)`，则 signal 和 thinkingLevel 被静默忽略。
- **影响:** compaction 无法通过 abort signal 中断。

### 15. 前端 API 端点不存在

- **文件:** `frontend/lib/api.ts:303`; `frontend/lib/hooks.ts:191-197,237`
- **现象:** `getAdvancedProposals()` 调用 `GET /api/projects/{id}/advanced/proposals`，后端 `advanced.py` 中无此路由。
- **影响:** `useWorkspaceRefresh` 每次刷新都触发 404，Proposal 同步缓存写入后 refetch 失败。

## 中危问题 (Medium)

### 16. Manager Agent 工具错误处理不一致

- **文件:** `manager-agent/src/server.js:878-1794`
- **现象:** 16 个工具（`find_cards`、`get_card_detail`、`inspect_project_summary` 等）未包裹 try-catch，网络错误直接抛给 Agent；15 个工具（`create_card`、`start_card_run` 等）正确使用 try-catch 返回 `toolErrorResult`。
- **影响:** 后端临时不可用时，部分工具调用会导致 Agent 对话中断，另一部分则优雅降级。

### 17. read_result_asset 缺少 encodeURIComponent

- **文件:** `manager-agent/src/server.js:1648`
- **现象:** asset_id 直接拼入 URL，未编码。对比同文件中 `get_asset_detail:989`、`get_card_detail:967` 等均正确使用了 `encodeURIComponent`。
- **影响:** 路径穿越风险。

### 18. SSE 事件可能写入已关闭的 Response 流

- **文件:** `manager-agent/src/server.js:301-303,2430-2444`
- **现象:** 客户端断开后 `abortController.abort()` 触发，但 agent 可能继续通过 emit 回调调用 `writeSseEvent`，向已销毁的 response 写入。
- **影响:** 未处理的写入异常。

### 19. 前端代理路由二进制文件上传损坏

- **文件:** `frontend/app/api/[...path]/route.ts:23`
- **现象:** `request.text()` 将 `multipart/form-data` 的二进制内容按 UTF-8 解码，非 UTF-8 字节被替换为 U+FFFD。同时缺少 `duplex: "half"`。
- **影响:** PNG、xlsx、zip 等二进制文件上传后损坏。注：该路由在 rewrite 之后实际为死代码（见 #26），但若 rewrite 被移除则会暴露此问题。

### 20. useRuntimeApprovals 轮询不随 runStatus 停止

- **文件:** `frontend/lib/hooks.ts:219`
- **现象:** `refetchInterval` 使用静态标量值，React Query 不在 run 状态变化后重新评估。对比 `useRunEvents`（第 204 行）正确使用了函数形式的 `refetchInterval`。
- **影响:** run 已结束后仍每 4 秒轮询，浪费带宽和后台资源。

### 21. deletingAssetId 并发删除时状态错乱

- **文件:** `frontend/components/files/FilesPanel.tsx:168,180,192`
- **现象:** `useMutation.variables` 只追踪最近一次 mutation。快速连续删除时，旧删除按钮的 disabled 状态提前恢复。
- **影响:** 可能导致重复删除请求。

### 22. 部署脚本环境变量重复写入

- **文件:** `scripts/deploy_user_systemd.sh:244-257`
- **现象:** heredoc 写入默认值后，`env | grep '^BLUEPRINT_'` 追加用户当前环境中的同名前缀变量。
- **影响:** systemd 采用 last-write-wins，用户环境变量静默覆盖脚本检测值，行为不可预测。

### 23. 后端 uvicorn 缺少 --proxy-headers

- **文件:** `deploy/systemd/blueprint-re-backend.service:9`
- **影响:** 在反向代理后部署时，客户端真实 IP 和协议信息丢失。

### 24. Dev 脚本端口与配置默认值不一致

- **文件:** `scripts/dev.sh:37`; `backend/app/core/config.py:32`
- **现象:** dev.sh 在 8000 端口启动后端，`backend_api_base_url` 默认值为 `http://127.0.0.1:18001/api`。
- **影响:** 未设置环境变量时 manager-agent 请求发送到错误的端口。

### 25. transformContext 在 compaction 启用时无裁剪效果

- **文件:** `manager-agent/src/server.js:2238-2239`
- **现象:** `MANAGER_COMPACTION_ENABLED` 为 true 时，`transformContext` 直接返回原始消息不做任何裁剪。
- **影响:** 长时间对话上下文持续增长，可能导致 token 超限。

### 26. 前端 API 代理路由是死代码

- **文件:** `frontend/app/api/[...path]/route.ts`
- **现象:** `next.config.ts` 的 rewrite 已在路由层拦截所有 `/api/*` 请求，该 catch-all route handler 永远不会被调用。
- **影响:** 无功能影响，但增加了代码维护负担。

### 27. 安装脚本未写入 library_summarizer_model

- **文件:** `scripts/install_blueprint_re.sh:162-195`
- **影响:** 用户无法通过安装脚本配置此选项，需手动编辑 `.env`。
